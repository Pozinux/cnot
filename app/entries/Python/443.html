L'avantage est que c'est une bdd sur le disque dans un fichier donc pas de connexions, d'installation d'un SGBD etc.
  
Il faut importer le package :
  
import sqlite3
  
Création de la base :
  
Quand on fait la connexion, il va créer le fichier s'il n'existe pas.
  
sqlite3.connect("database_folder/database_db_file.db")
  
On récupère cette connexion dans une variable "conn" :
  
conn = sqlite3.connect("database_folder/database_db_file.db")
  
Il faudra penser à fermer cette connexion à la fin :
  
conn.close()
  
Créer une table :
  
Pour cela il faut créer un curseur.
  
c = conn.cursor()
  
C'est avec ce curseur que l'on exécute les requêtes sql.
  
Pour exécuter une requête :
  
Ici la requête sql créé une table et ses colonnes.
  
c.execute("""
CREATE TABLE table_test(
    col_1 text,
    col_2 text    
""")
  
Il faut ensuite faire un COMMIT, qui veut dire à notre bdd que l'on veut y ajouter ce que l'on a dans notre requête. Tant que l'on ne l'a pas fait, c'est mis dans une sorte de stack mais ce n'est pas exécuté tant qu'on a pas fait le commit. Tout ça est juste en mémoire mais n'est pas effectif dans la bdd. On peut d'ailleurs voir un fichier temporaire créé "-journal" :
  
  
conn.commit()
  
Dans cette requête, si la table existe déjà, il va faire une erreur, donc on précise de le faire que si la table n'existe pas déjà :
  
c.execute("""
CREATE TABLE IF NOT EXISTS table_test(
    col_1 text,
    col_2 text 
)   
""")
  
Ajouter des données :
  
On peut aussi simplement le faire avec "SQLiteDatabaseBrowser" ->  SQLiteDatabaseBrowser
  
Mais sinon en Python :
  
dictionnaire_1 = {"serveur_name_key": "VM_3", "dns_name_key": "VM_1.local", "host_name_key": "ESXi_1", "management_name_key": "vCenter_1"}
  
c.execute("INSERT INTO serveur_vmware VALUES (:serveur_name_key, :dns_name_key, :host_name_key, :management_name_key)", dictionnaire_1)
  
Note : Comme pour MYSQL, il existe aussi une fonction "executemany()" :
  
  
Récupérer des données 
  
c.execute("SELECT * FROM serveur_vmware WHERE serveur_name='VM_3'")
result = c.fetchall()
print(result)
  
RESULTAT :
  
[('VM_3', 'VM_1.local', 'ESXi_1', 'vCenter_1')]
  
Avec Like :
  
c.execute("SELECT * FROM serveur_vmware WHERE serveur_name LIKE \'%VM%\'")
  
RESULTAT :
  
[('vm_1', 'dns_name_1', 'esxi_1', 'vcenter_1'), ('vm_2', 'dns_name_2', 'esxi_1', 'vcenter_1')]
  
On peut aussi passer un dictionnaire :
  
dico = {"key_1": "vm_2"}
c.execute("SELECT * FROM serveur_vmware WHERE serveur_name=:key_1", dico)
  
RESULTAT :
  
[('vm_2', 'dns_name_2', 'esxi_1', 'vcenter_1')]
  
On peut aussi utiliser le dico pour sélectionner plusieurs données :
  
dico = {"serveur_name_key": "vm_1", "management_name_key": "vcenter_2"}
c.execute("SELECT * FROM serveur_vmware WHERE serveur_name=:serveur_name_key OR management_name=:management_name_key ", dico)
  
RESULT :
  
[('vm_1', 'dns_name_1', 'esxi_1', 'vcenter_1'), ('vm_2', 'dns_name_2', 'esxi_1', 'vcenter_2')]
  
Note sur le cursor 
  
La méthode fetchall() permet de récupérer tout ce qui est sélectionné par la requête sql.
C'est ce que l'on appelle un générateur. 
Le générateur, une fois que l'on va avoir récupéré les données de la requête, il va être épuisé. C'est à dire que l'on ne peut pas le refaire une deuxième fois.
Le curseur, c'est comme pour les fichiers, quand on l'a lu, il se retrouve à la fin. Il faut replacer le curseur au début du fichier pour pouvoir le relire. C'est pareil pour les bases de données.
Si on exécute deux fois un fetchall() sur la même requête sql et que l'on print le résultat, le deuxième sera vide. 
 
Mettre à jour des données :
  
new_dico = {"serveur_name_key": "vm_1", "management_name_key": "vcenter_3"}
c.execute("UPDATE serveur_vmware SET management_name=:management_name_key WHERE serveur_name=:serveur_name_key", new_dico)
  
Supprimer des données :
  
c.execute("DELETE FROM serveur_vmware WHERE serveur_name='vm_1'")
  
Attention, si on ne met pas de WHERE, on supprime toutes les entrées de la table :
  
c.execute("DELETE FROM serveur_vmware")
  
Tout cela donne :
  
import sqlite3
  
conn = sqlite3.connect("data/sqlite_db_file.db")
c = conn.cursor()
  
# Créer les tables
c.execute("""
CREATE TABLE IF NOT EXISTS serveur_vmware(
    serveur_name text,
    dns_name text,
    host_name text,
    management_name text  
)
""")
c.execute("""
    CREATE TABLE IF NOT EXISTS serveur_opca(
        serveur_name text,
        dns_name text,
        host_name text,
        management_name text  
    )
    """)
  
# Ajouter des données
dictionnaire_1 = {"serveur_name_key": "VM_3", "dns_name_key": "VM_1.local", "host_name_key": "ESXi_1", "management_name_key": "vCenter_1"}
c.execute("INSERT INTO serveur_vmware VALUES (:serveur_name_key, :dns_name_key, :host_name_key, :management_name_key)", dictionnaire_1)
  
# Récupérer des données 
c.execute("SELECT * FROM serveur_vmware WHERE serveur_name='VM_3'")
result = c.fetchall()
print(result)
  
# Récupérer des données avec LIKE
c.execute("SELECT * FROM serveur_vmware WHERE serveur_name LIKE \'%VM%\'")
  
# Récupérer avec un dico
dico = {"serveur_name_key": "vm_1", "management_name_key": "vcenter_2"}
c.execute("SELECT * FROM serveur_vmware WHERE serveur_name=:serveur_name_key OR management_name=:management_name_key ", dico)
  
# Mettre à jour des données
new_dico = {"serveur_name_key": "vm_1", "management_name_key": "vcenter_3"}
c.execute("UPDATE serveur_vmware SET management_name=:management_name_key WHERE serveur_name=:serveur_name_key", new_dico)
  
# Supprimer des données
c.execute("DELETE FROM serveur_vmware WHERE serveur_name='vm_1'")
  
# Commiter les requêtes + fermer la bdd (toujours !)

conn.close()
