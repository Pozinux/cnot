GRUB2 arrive avec RHEL7. Utiliser les commandes pour modifier. Ne pas le faire manuelle dans les fichiers de conf.
                                   
Deux moyens de lister les entrées kernels que l'on verra dans le Grub :                                                
                                                  
# awk -F\' '$1=="menuentry " {print $2}' /etc/grub2.cfg

Relax-and-Recover

Vérifier laquelle entrée est par défaut :
                                      
Note : uname -a indique sur lequel il a dernièrement booté.                                           
                                      
# grub2-editenv list                                           
saved_entry=Oracle Linux Server (4.14.35-1902.7.3.1.el7uek.x86_64 with Unbreakable Enterprise Kernel) 7.2                                           
                                         
 ou sur quel kernel il va booter par défaut dans /boot :
                                      
# grubby --default-kernel                                      
/boot/vmlinuz-4.14.35-1902.7.3.1.el7uek.x86_64                                      
                                      
 
Ajouter ou supprimer des entrées :
                               
Use yum or rpm to add or remove installed kernels - Don’t go toying around manually - You will mess things up and cost yourself time.
                               
                             
Changer l'entrée par défaut :
                             
grub2-set-default 0    
                             
Aussi, la commande suivante sélectionnera ce kernel spécifique s'il existe :
                             
grub2-set-default ‘Fedora Linux, with Linux 3.1.2-1.fc16.x86_64’
                             
 
Resettings GRUB2
                          
Parfois ça ne fonctionne plus il faut reseter :
                          
rm /etc/grub.d/* (delete all grub.d scripts)                          
rm /etc/sysconfig/grub (remove all system configuration)                          
yum reinstall grub2-tools (reinstall the entire package)                          
grub2-mkconfig -o /boot/grub2/grub.cfg (reset the entire configuration)
                            
                          
Reinstalling GRUB2
  
This will reinstall and restore any corrupted files in the /boot/grub2/ directory - If files are missing, they will be recreated.
                          
grub2-install <device>
  
Voir exemple de cas perso ici :
          
Linux ne boot pas - écran noir ou parfois booting system... mais ne boot pas - Réinstallation de grub
                            
Finaliser les changements
  
IF YOU CHANGE ANYTHING IN GRUB, YOU NEED TO RUN grub2-mkconfig OR THE CHANGES WILL NOT STICK.
                       
Si seulement on est sur un BIOS traditionnel :
                          
grub2-mkconfig -o /boot/grub2/grub.cfg
                            
Si seulement on est sur un BIOS UEFI :
 
grub2-mkconfig -o /boot/efi/EFI/redhat/grub.cfg
  
                          
Telling GRUB to boot into different targets
     
Move into the append/edit mode with the e key at boot menu
                       
IF USING x86-64 -- append boot directive to the end of the linux16 line
IF USING UEFI -- append boot directive to the end of the linuxefi line
                       
Par exemple, ajouter :
                       
systemd.unit=[systemctl.target]
