  • Clone OK mais hot clone risky -> Voir Doc Oracle ID 2221697.1 :
"The hot clone is the only option to keep the VM running and take a snapshot - But it is not guaranteed to be able to be booted after the hot clone has been taken. This is why each hot clone should be checked for data integrity and consistency.
The hot clone operation will create a copy of the disk - The problem of the hot cloning procedure above while the VM is running does not guarantee the disk will be in a consistent state when it will be reattached to a Virtual Machine as there is constant I/O going on, therefore to have a guaranteed consistent backup, the VM needs to be stopped first.
To have a guaranteed data-consistent "snapshots" without the need to check the backup's validity, stop the virtual machine, clone its disks or the whole VM and start again the VM."
