Attention le .gitignore ne sert que pour les fichier qui ne sont pas trackés (que je n'ai pas ajouté avec add .)
J'ai besoin que le fichier credentials.php soit "tracké" car il faut qu'il existe en remote mais pas que les changements que je lui fais en local soient "pushés" donc je lui met un flag git "--skip-worktree". Comme ça, même si je le modifie en local, lorsque je fais un git status il ne me dira pas qu'il y a eu des modifs. Pour mettre et enlever des flags à des fichiers et voir quels fichiers ont des flags (le "H" signifie pas de flag) :
  
[root@vps614515 cnot]# git ls-files -v | grep creden 
H app/credentials.php 
  
[root@vps614515 cnot]# git update-index --assume-unchanged app/credentials.php  
  
[root@vps614515 cnot]# git ls-files -v | grep creden 
h app/credentials.php 
 
[root@vps614515 cnot]# git update-index --no-assume-unchanged app/credentials.php 
  
[root@vps614515 cnot]# git ls-files -v | grep creden 
H app/credentials.php 
  
[root@vps614515 cnot]# git update-index --skip-worktree app/credentials.php 
  
[root@vps614515 cnot]# git ls-files -v | grep creden  
S app/credentials.php
